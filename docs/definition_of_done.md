# Definition of Done (DoD) — CarRent

Une User Story est Done uniquement si elle
respecte ces 6 critères :

| # | Critère | Dimension | Vérification |
|---|---------|-----------|--------------|
| 1 | Code revu par un autre développeur | Qualité code | Pull Request approuvée |
| 2 | Tests unitaires passent à 100% | Tests | Pipeline CI/CD |
| 3 | Fonctionnalité déployée sur recette | Déploiement | Build Expo EAS |
| 4 | Validation du Product Owner | Validation PO | Compte-rendu Review |
| 5 | Aucun bug bloquant sur Jira | Qualité produit | Tableau Jira |
| 6 | Documentation mise à jour | Documentation | GitHub README |

## Règle
Une User Story ne peut être considérée comme
terminée que si TOUS les critères sont respectés.
